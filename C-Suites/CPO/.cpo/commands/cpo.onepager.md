### Execution Flow

1. **Gather Initiative Information**
   - If user provided input, extract initiative details
   - If starting fresh, ask canonical questions

2. **Apply PM Constitution Checks**
   - Is the problem clear? (If weak, push back)
   - Is there a measurable outcome hypothesis?
   - Is this the smallest testable bet?

3. **Populate One-Pager**
   - Fill all sections with available information
   - Mark sections needing input as [NEEDS INPUT]
   - Flag risks for appropriate C-suite coordination

4. **RICE/WSJF Scoring**
   - Calculate priority score
   - Compare to other backlog items if known

5. **Log Decision**
   - Add to decision log
   - Save one-pager

## Quality Checklist

Before finalizing, verify:

- [ ] Problem statement is specific and validated
- [ ] Target user is clearly defined
- [ ] Success metrics are measurable
- [ ] Scope has explicit "out" items
- [ ] Risks are identified with mitigations
- [ ] Single accountable owner assigned
- [ ] Tradeoffs are explicit
